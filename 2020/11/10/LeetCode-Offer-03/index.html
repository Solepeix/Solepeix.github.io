找出数组中重复的数字。

在一个长度为 n 的数组 nums 里的所有数字都在 0～n-1 的范围内。数组中某些数字是重复的，但不知道有几个数字重复了，也不知道每个数字重复了几次。请找出数组中任意一个重复的数字。

示例 1：

输入：

[2, 3, 1, 0, 2, 5, 3]

输出：2 或 3

思路一

使用哈希表存储数组出现的数字于次数

class Solution {

public int findRepeatNumber(int[] nums) {

​ HashMap<Integer,Integer>map=new HashMap<>();//创建哈希表存放出现的次数

​ int i=0;//从数组下标0开始遍历

​ while(i<nums.length){

​ if(!map.containsKey(nums[i])){//判断是否有nums[i]存在

​ map.put(nums[i],1);//没有的话放入

​ i++;//指针移动

​ }else{

​ return nums[i];//已经有的话直接返回

​ }}

​ return -1;

}

}

思路二

由于互相不重复，使用Set数据结构

class Solution {

public int findRepeatNumber(int[] nums) {

​ HashSetset=new HashSet<>();//创建哈希表存放出现的次数

​ int i=0;//从数组下标0开始遍历

​ while(i<nums.length){

​ if(!set.contains(nums[i])){//判断是否有nums[i]存在

​ set.add(nums[i]);//没有的话放入

​ i++;//指针移动

​ }else{

​ return nums[i];//已经有的话直接返回

​ }}

​ return -1;

}

}

优化思路：减小空间复杂度，因为之前的空间复杂度开辟了哈希表

原地置换空间复杂度为O(1)

思路三

由于长度为 n 的数组 nums 里的所有数字都在 0～n-1 的范围内

如果没有重复数字，那么正常排序后，数字i应该在下标为i的位置，所以思路是重头扫描数组，遇到下标为i的数字如果不是i的话，（假设为m),那么我们就拿与下标m的数字交换。在交换过程中，如果有重复的数字发生，那么终止返回ture

class Solution {

public int findRepeatNumber(int[] nums) {

​ int temp=0;

​ for(int i=0;i<nums.length;i++){

​ while(nums[i]!=i){

​ if(nums[i]==nums[nums[i]]){

​ return nums[i];//存在重复的数字

​ }

​ temp=nums[i]; //进行数字交换，因为一个萝卜一个坑，如果没有重复数字，肯定可以换到正确的位置

​ nums[i]=nums[temp];

​ nums[temp]=temp;

​

​ }

​ }

​ return -1;

}

}

思路四

使用临时数组存放出现次数

class Solution {

public int findRepeatNumber(int[] nums) {

​ //新建一个临时数组存放各个数字出现的次数

​ int [] temp =new int [nums.length];

​ for(int i=0;i<nums.length;++i){

​ temp[nums[i]]++;

​ //数字出现次数大于1直接返回

​ if(temp[nums[i]]>1){

​ return nums[i];

​ }

​ }

​ return -1;

}

}
